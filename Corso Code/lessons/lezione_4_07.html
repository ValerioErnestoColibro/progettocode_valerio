Lezione 4/07

  • Le chiamate API (Application Programming Interface) permettono ad un client di parlare con un server;
  • Il server espone diverse API attraverso le quali comunica;
  • Le API sono gli indirizzi che io digito;
  • I metodi delle API sono questi:
    • Get: chiamata di base a cui succede una pagina HTML;
    • Post: per rispondere ha bisogno di un'informazione (esempio: la funzione "login" ha bisogno di nomeutente e password);
    • Patch: si usa per le modifiche per i campi nuovi (desidera SOLO alcuni parametri);
    • Put: si usa per le modifiche globali (desidera tutti i parametri);
    • Delete: si usa per eliminare dei dati;
  • Status Code: sono stati di risposta (esempio: "Non autorizzato") che influenzano i motori di ricerca;
  • Alcune informazioni le salveremo in un database;
  • Per fare una chiamata asincrona, la sintassi è:

    async function abc() {

    const res = await nomeFunzioneAsincrona; //"Await" serve per attendere l'esecuzione di una funzione asincrona

    return res

    }

  • Per migliorarne la leggibilità, si scrive una funzione asincrona in un modo sincrono (dal basso verso l'alto);
  • "Fetch" è una funzione di tipo asincrono, e permette di mettersi in lista per visualizzare il sito:

    fetch("www.google.com")

  • Se una funzione ritorna una promessa, posso raccogliere il risultato della funzione tramite "Async Await" oppure

    fetch ("www.google.com").then()

  • L'inferno delle callback (Callback Hell) prevede l'uso di ".catch()" per restituire un errore;
  • Consigli sulla pianificazione dell'attività:
    1. Capire quale progetto dovrà essere fatto;
    2. Creare una lista delle cose da fare;
    3. Capire cosa serve per farle;
    4. Capire quanto tempo prende ogni cosa da fare;
    5. Valutare la difficoltà di ogni cosa da fare in maniera numerica (scram poker, ispirato alla gamification);
  • Creare un file .json con una classe, al suo interno il login e le parole "Path", "Method", "Authenticated: true/false"
  • Il formato "swagger" è una libreria che permette di documentare l'API (da cercare);
