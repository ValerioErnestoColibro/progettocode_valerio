Prima del cloud computing

  • Lo spazio era insufficiente;
  • Bisognava fare una stima dei costi MOLTO accurata;


Ma cosa è il cloud computing?

  • Distribuzione on-demand
  • Nessun investimento significativo;
  • Provisioning (verifica delle capacità di calcolo) delle risorse informatiche secondo necessità;
  • Pagamento SOLO in base all'uso effettivo;


Componenti del cloud coumputing

  • Server;
  • Archiviazione;
  • Database;
  • Applicazioni;


Modelli di distribuzione del cloud

  • Full Cloud (iniziare a progettare da lì);
  • Ibrido (tipico scenario aziendale);
  • On-premise (cloud privati);


Vantaggi di AWS

  • Agilità;
  • Elasticità;
  • Risparmio sui costi;
  • Distribuzione rapida;


Agilità

  • Più innovazione;
  • Non bisogna attendere altre risorse;


Elasticità

  • Adattamento per far fronte ai carichi di lavoro variabili;
  • Distribuzione dei carichi di lavoro in più risorse;
  • Distribuzione di risorse e applicazioni con tolleranza ai guasti;


Risparimio sui costi

  • Sostituzione delle spese in conto capitale con spese variabili;
  • Riduzione dei costi di manutenzione;
  • Pagamento in base al consumo;


Distribuzione rapida

  • Piattaforma altamente scalabile a livello mondiale;
  • Distribuzione automatizzata;
  • Rilascio rapido di nuove caratteristiche nelle applicazioni;


Infrastruttura globale

  • Decine di paesi in tutte le zone del mondo;
  • 105 zone di disponibilità in molte regioni;


Regioni

  • Una Regione è una posizione geografica distinta con più zone isolate l'una dall'altra:
    • Supporta la presenza globale di AWS;
    • Serve i clienti su scala globale;
    • Rapida espansione con un impegno a lungo termine per la sostenibilità;


Zone di disponibilità

  • Parti completamente isolate dell'infrastruttura globale AWS;
  • Le zone sono connesse grazie all'elevata larghezza di banda e bassa latenza grazie alla fibra dedicata completamente ridondante;
  • Aiutano i clienti a costruire applicazioni altamente disponibili;
  • Contengono i "Data Center", immense stanze con computers e cablaggi;


Punti di presenza

  • Insieme di edge location e server di cache edge regionali;
  • Forniscono contenuti ai clienti tramite una rete globale di posizioni dei punti di presenza;
  • Bassa latenza ed elevata velocità di trasferimento;


Vantaggi di AWS (riassunto)

  • Agilità;

Servizi nel dettaglio

  • Amazon EC2:
    • Capacità di calcolo sicura e scalabile;
    • Consente di avviare istanze del server in pochi minuti, e dimensionare la capacità a seconda delle esigenze;
    • Varie opzioni di prezzi per ridurre i costi;
    • Istanze on-demand, spot e riservate;
  • Amazon EC2 Auto Scaling:
    • Aggiunta o rimozione automatica delle istanze EC2 in base alle condizioni definite;
    • Capacità di preservare la disponibilità delle applicazioni;
    • Vantahhi derivanti dalla scalabilità dinamica e predittiva;
    • Aggiunta di nuove istanze SOLO se necessario;
    • Rimozione delle istanze superflue quando il traffico è ridotto;
    • Pagamento SOLO in base all'uso;
  • Elastic Load Balancing:
    • Distribuzione automatica del traffico tra più risorse;
    • Singolo punto di contatto per il gruppo Auto Scaling;
    • Distribuzione del traffico tra più zone di disponibilità;
    • Application Load Balancer: utilizzato per il bilanciamento del carico di traffico HTTP e HTTPS, funziona a livello di richiesta individuale (livello 7), e instrada il traffico in base al contenuto di una richiesta;
  • Amazon Elastic Container Service (ECS):
    • Servizio di orchestrazione di container con scalabilità e prestazioni elevate;
    • Uso di chiamate API semplici per avviare e arrestare applicazioni compatibili con Docker;
  • AWS Lambda:
    • Esecuzione del codice senza effettuare il provisioning o gestire i server;
    • Pagamento solo per il tempo d'uso;
    • uso di altri servizi AWS per attivare in automatico il codice;


Archiviazione

  • Amazon simple storage service (Amazon S3);
  • Amazon Elastic Block Store;


Amazon Simple Storage Service (S3)

  • Consente di archiviare e accedere a qualsiasi tipo di dato su Internet;
  • Alta durabilità;
  • Include una vasta gamma di classi di archiviazione convenienti nei costi;
  • Archiviazione di file in bucket;
  • Supporto di una vasta gamma di casi d'uso;
  • Possibilità di tenere traccia di più varianti di un oggetto tramite versioni multiple;
  • Capacità di evitare eliminazioni accidentali degli oggetti;


Amazon Elastic Block Store (EBS)

  • Archiviazione a blocchi ad alte prestazioni;
  • Disponibiltà altissima;
  • Dimensionamento verso l'alto/il basso in pochi minuti;
  • Quattro tipi di volume per ottimizzare prezzo e prestazioni: SSD per uso generico, SSD con provisioning IOPS, HDD ottimizzato per il througput, Cold HDD;


Amazon Elastic File System (EFS)

  • File system semplice, scalabile e tutto gestito;
  • Capacità di dimensionamento on-demand fino ai petabyte, senza interrompere le applicazioni;
  • Archiviazione dei dati su base regionale per agevolare la disponibilità;
  • Elasticità dinamica;
  • Archiviazione di file condivisa;
  • Costi contenuti;


Database

  • Amazon Relation Database Service (Amazon RDS): servizio di database relazionale, automatizzazione delle attività di amministrazione, sicurezza e compatibilità, semplicità di amministrazione, uso di repliche di lettura per carichi di lavoro di database gravosi in lettura,;
  • Amazon DynamoDB: database documentale e chiave di valore NoSQL rapido e flessibile, nessun server da gestire, prestazioni al millisecond a una cifra su qualsiasi scala;
  • AWS Database Migration Service (AWS DMS): Migrazione dei database in AWS con rapidità e sicurezza, migrazione dei dati da e verso i database più usati, piena operatività dei database di origine durante la migrazione;
  • Applicazioni web serveless
