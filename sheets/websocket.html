WebSocket


Definizione

WebSocket è una tecnologia che fornisce una comunicazione bidirezionale full-duplex tra client e server su una singola connessione TCP. Questo protocollo è stato standardizzato come RFC 6455 ed è progettato per essere utilizzato nelle applicazioni web moderne che richiedono aggiornamenti in tempo reale, come chat, giochi online, applicazioni di collaborazione in tempo reale e altro ancora.

Ecco una panoramica dei principali concetti e funzionalità dei WebSocket


Connessione

La connessione WebSocket inizia con una handshake HTTP, che viene poi "promossa" a una connessione WebSocket. Durante questo handshake, il client invia una richiesta HTTP con l'intestazione Upgrade: websocket, indicando al server di passare al protocollo WebSocket.


Comunicazione Bidirezionale

Una volta stabilita la connessione, sia il client che il server possono inviare dati in qualsiasi momento, senza dover riaprire la connessione. Questo è in contrasto con il tradizionale modello HTTP, dove il client deve fare una richiesta e il server risponde.


Efficienza

WebSocket utilizza un protocollo binario che riduce l'overhead rispetto al polling HTTP, rendendolo più efficiente per applicazioni che richiedono aggiornamenti frequenti e in tempo reale.


Supporto nei browser

La maggior parte dei moderni browser web supporta WebSocket, rendendolo una scelta pratica per le applicazioni web.


Esempio di utilizzo in Javascript

let socket = new WebSocket("ws://esempio.com/socketserver"); // Creare una nuova connessione WebSocket

socket.onopen = function(event) { // Event handler per quando la connessione è aperta

socket.onopen = function(event) {

console.log("Connessione aperta");

socket.send("Ciao, server!");

};


socket.onmessage = function(event) { // Event handler per quando arriva un messaggio

console.log("Messaggio ricevuto: " + event.data);

};


socket.onclose = function(event) { // Event handler per quando la connessione è chiusa

console.log("Connessione chiusa");

};


socket.onerror = function(error) { // Event handler per quando si verifica un errore

console.error("Errore: " + error.message);

};


Vantaggi dei WebSocket:

  • Bassa Latenza: Essendo una connessione persistente, i WebSocket riducono la latenza rispetto ai metodi di polling tradizionali.
  • Efficienza: Meno overhead di rete rispetto a HTTP.
  • Comunicazione in Tempo Reale: Ideale per applicazioni che richiedono comunicazioni rapide e frequenti.


Svantaggi dei WebSocket:

  • Compatibilità: Anche se la maggior parte dei browser moderni supporta WebSocket, ci possono essere problemi di compatibilità con browser o ambienti molto vecchi.
  • Sicurezza: Come tutte le connessioni aperte, possono essere vulnerabili agli attacchi se non adeguatamente protetti (es. con l'uso di WSS, il WebSocket su TLS).


Utilizzo in Applicazioni Web:

  • Chat e Messaggistica: Applicazioni di messaggistica in tempo reale come Slack o WhatsApp Web.
  • Gaming Online: Giochi multiplayer online che richiedono aggiornamenti rapidi e costanti.
  • Collaborazione in Tempo Reale: Strumenti di collaborazione come Google Docs o applicazioni di editing collaborativo.
  • Aggiornamenti in Tempo Reale: Applicazioni di trading, dashboard di monitoraggio, notifiche push e altro ancora.
